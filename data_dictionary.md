# Data Dictionary

HumanLoop Command Center uses deterministic synthetic data generated by `backend/app/seed.py`.

| Table | Purpose | Important Fields |
|---|---|---|
| `customers` | AI lab or enterprise customer accounts. | `name`, `segment`, `contact_email`, `created_at` |
| `projects` | Customer intake record and delivery target. | `domain`, `task_type`, `target_dataset_size`, `deadline`, `quality_threshold`, `required_expertise`, `priority`, `delivery_format` |
| `contributors` | Expert labelers and reviewers available for assignment. | `domain_expertise`, `capacity_per_week`, `approval_rate`, `average_rubric_score`, `rejection_rate`, `training_status`, `coaching_flag`, `historical_throughput` |
| `contributor_training` | Calibration and training module progress. | `module_name`, `status`, `completed_at` |
| `evaluation_tasks` | Individual prompt-response evaluation units. | `prompt`, `status`, `qa_flag_count`, `assigned_contributor_id`, `reviewer`, `submission_time`, `risk_score` |
| `model_responses` | Synthetic model answers being reviewed. | `model_name`, `response`, `response_version` |
| `human_ratings` | Human label/review events, including final and secondary review. | `rating`, `rubric_score`, `reviewer_comments`, `review_event_type`, `submission_seconds`, `is_final` |
| `rubric_scores` | Structured scoring dimensions used by QA and analytics. | `criteria_accuracy`, `criteria_completeness`, `criteria_safety`, `criteria_reasoning`, `overall_score`, `consistency_check` |
| `qa_reviews` | QA decision and rejection reason per task. | `qa_status`, `notes`, `reviewed_at`, `rejection_reason` |
| `quality_flags` | Validation issues surfaced by the quality control engine. | `category`, `severity`, `owner`, `status`, `description` |
| `exports` | Audit log of dataset exports. | `format`, `row_count`, `checksum`, `created_at` |
| `activity_logs` | Operational event stream for seed and project actions. | `entity_type`, `entity_id`, `action`, `actor`, `detail` |

## Generated Scale

- 40 customer projects
- 300 expert contributors
- 12,000 evaluation tasks
- 25,000 human rating/review events
- 5 domain rubric families represented across 6 task domains
- 1,000+ QA and operational quality flags

## Quality Flag Categories

| Category | Meaning |
|---|---|
| `missing_rating` | A submitted task has no final human rating. |
| `duplicate_prompt` | Prompt text repeats a prior task and needs deduplication review. |
| `inconsistent_scoring` | Reviewer scores diverge beyond allowed rubric variance. |
| `low_effort_comment` | Reviewer comment is too short for customer auditability. |
| `contributor_drift` | Contributor quality is moving away from historical baseline. |
| `fast_submission` | Labeling duration is unusually short for task complexity. |
| `high_rejection_contributor` | Contributor rejection rate exceeds coaching threshold. |
| `deadline_risk` | Project deadline is near while readiness is insufficient. |
| `quality_threshold` | Task or project falls below customer quality threshold. |
